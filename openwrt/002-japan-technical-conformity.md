# OpenWrtを日本で使う前に｜Wi-Fiルーターと技適の話をざっくり整理する【OpenWrt集中連載002】

OpenWrtに興味がある人なら、一度はこう思うはずです。

「対応機種を買ってOpenWrtを入れれば、自分好みの最強ルーターが作れるのでは？」

分かります。

LuCIも使える。  
SSHも入れる。  
opkgで機能も足せる。  
VLANもVPNもDNSも細かく触れる。

ネットワーク好きからすると、かなり楽しそうです。

ただ、Wi-Fiルーターの場合は、ここで少しだけ立ち止まったほうがいいです。

有線ルーターや検証用の小型PCなら、「動くかどうか」がまず大きなポイントになります。

でもWi-Fiルーターは、電波を出す機器です。

つまり、日本国内で使う場合は、

```txt
OpenWrtが動くか
```

だけではなく、

```txt
日本国内向けの無線機器として安心して使えるか
```

も見ておきたい、という話になります。

この記事は、法律の細かい解釈をする記事ではありません。

技適の条文を読み込むというより、

「なぜ日本でOpenWrt系Wi-Fiルーターを使う時に、国内向け製品かどうかを見たほうがいいのか」

を、ざっくり現実的に整理します。

> この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使いながら、家庭・小さなオフィス・店舗でOpenWrtベースルーターを現実的に使う方法をまとめています。

## 先にざっくり結論

日本でOpenWrt系Wi-Fiルーターを使う時は、次の3つを分けて考えると分かりやすいです。

1. 技術的にOpenWrtが動くか
2. Wi-Fi機器として国内向けにどう案内されているか
3. 家庭・小さなオフィス・店舗で安心して運用に乗せられるか

OpenWrtの対応機種リストや海外の導入記事は、1を見るにはとても役に立ちます。

ただ、日本国内でWi-Fiルーターとして使うなら、2と3もかなり大事です。

この連載でLN6001-JPを題材にするのは、ユーザーが一般市販ルーターへ非公式ファームウェアを書き込む前提ではなく、OpenWrtベースの国内向けWi-Fi 7ルーターとして出荷されている製品だからです。

ここがかなり大きいです。

もちろん、この記事は個別の法律判断ではありません。

購入時や業務利用時には、製品ページ、公式サポート、本体ラベル、販売代理店の情報を確認してください。

不安がある場合は、メーカーや販売代理店へ確認するのが安全です。

## まず言葉だけそろえる

最初に、ざっくり用語をそろえておきます。

![表画像 table-01](https://raw.githubusercontent.com/ikm-san/blog/main/openwrt/assets/002/table-01.png)

ここで大事なのは、OpenWrtの自由度と、Wi-Fi機器としての国内利用前提を分けて考えることです。

VLANやDNSやVPNを触る話と、無線の国設定や送信出力を触る話は、同じ「設定」でも重みが違います。

## 技適の話は、なぜWi-Fiルーターで出てくるのか

Wi-Fiルーターは、小さなコンピューターであると同時に、電波を出す無線機器です。

ここがポイントです。

PCにLinuxを入れる話とは少し違います。

Wi-Fiルーターでは、次のような要素が国や地域の制度と関係します。

- 使用できる周波数帯
- 使用できるチャネル
- 送信出力
- DFS
- 国・地域ごとの無線制御
- 6GHz帯の扱い
- 最大通信速度の表記

OpenWrtの設定や無線ドライバ、ファームウェアは、こうした無線制御にも関係します。

海外のOpenWrt記事を見ると、国設定、送信出力、チャネル幅、DFS、6GHz帯の扱いなどを細かく調整する例が出てくることがあります。

読むぶんには面白いです。

でも、日本国内で家庭や店舗のWi-Fiルーターとして使う場合、そこを「面白そうだから触ってみる」対象にするのはおすすめしません。

ここはチューニング項目というより、

```txt
日本国内で電波を出すための前提
```

として扱うほうが安全です。

## 「動く」と「使ってよい」は別の話

OpenWrtの世界では、

```txt
この機種にOpenWrtを入れられた
このチップセットで動いた
LuCIもSSHも使えた
```

という情報がとても価値を持ちます。

これは技術的な検証として、本当に大事です。

ただし、家庭・小さなオフィス・店舗で使うWi-Fiルーターを選ぶなら、見る軸はもう少し増えます。

## 1. 技術的に動くか

OpenWrtがその機種に対応しているか。  
起動するか。  
Wi-FiやLANが使えるか。  
LuCIやSSHが使えるか。

これはOpenWrt導入の入口です。

## 2. 国内向け製品としてどう案内されているか

日本で販売されている型番なのか。  
国内向けページがあるのか。  
技適や国内法令への適合が案内されているのか。  
本体ラベルで確認できるのか。

Wi-Fiルーターの場合、この部分を飛ばすと不安が残ります。

## 3. 実用に乗せられるか

家庭や仕事で使うなら、毎回ファームウェアの書き換えや復旧手順から始めるのは大変です。

壊れた時に戻せるか。  
管理画面へ入れるか。  
初期設定が分かるか。  
サポート情報があるか。

ここもかなり大事です。

OpenWrtは面白いです。

でも、家庭や小規模環境では「面白い」だけでなく、

```txt
戻せる
説明できる
安定して使える
```

ことも同じくらい大切です。

## この連載で扱わないこと

この連載では、一般的な市販Wi-Fiルーターへ非公式ファームウェアを導入する手順は扱いません。

OpenWrtそのものを否定したいわけではありません。

むしろ逆です。

OpenWrtの柔軟性、LuCI、SSH、opkg、VLAN、VPN、DNS管理はかなり有用です。

ただ、日本国内でWi-Fiルーターとして使うなら、単に「OpenWrtが入るか」だけでなく、国内向け製品としてどう扱えるかも大事です。

そのため、この連載ではLN6001-JPを使います。

一般市販ルーターに非公式ファームウェアを入れる世界は、検証や学習としては深くて楽しいです。

でも、家庭・小さなオフィス・店舗で実用する記事としては、国内向けに整理されたOpenWrtベース製品を題材にしたほうが、読者に勧めやすいと考えています。

## LN6001-JPの立ち位置

LN6001-JPは、Linksys Velop WRT Pro 7の日本向けモデルです。

Linksys公式サポートでは、LN6001-JPについて、OpenWrtベースのWi-Fi 7トライバンドルーターであり、LuCIブラウザ管理画面、SSHログイン、opkgパッケージ、各種拡張機能に対応すると案内されています。

また、同じ公式サポートページでは、製品型番LN6001-JPについて「技適認証、国内法令適合」と案内されています。

さらにFAQでは、国内法に適合するために無線電波出力等の仕様を一部調整した日本国内専用モデルであり、電波出力の調整や地域コードの変更はできない、と説明されています。

ここが、一般的なOpenWrt導入と大きく違います。

ユーザーが自分でファームウェアを書き換えて、

```txt
これ、日本向けの無線制御として大丈夫なのかな？
```

と悩むところから始めるのではなく、OpenWrtベースの国内向け製品として使い始められます。

ここがLN6001-JPの大きな強みです。

## LN6001-JPは「OpenWrtを入れる製品」ではない

ここ、けっこう大事です。

LN6001-JPは、購入後にユーザーが自分でOpenWrtをインストールする製品ではありません。

OpenWrtベースのファームウェア、LuCI、SSH、初期SSID、初期パスワードなどが用意された状態で出荷される製品です。

つまり、最初にやることは、

```txt
OpenWrtを導入する
```

ではなく、

```txt
OpenWrtベースの環境をどう使うか
```

です。

通常のOpenWrt導入では、次のような作業が必要になりがちです。

- 対応機種を調べる
- ファームウェアを探す
- 書き換え手順を読む
- 失敗時の復旧方法を調べる
- 初期設定を自分で組む
- 無線まわりの扱いに不安が残る

LN6001-JPでは、この入口の負担がかなり軽くなります。

家庭や小さなオフィスで使うなら、この差はかなり大きいです。

## 買う前に確認したいこと

技適や国内利用の話は、法律用語として深掘りし始めると急に難しくなります。

ただ、購入前のチェックとしては、まず次を見るだけでもかなり実用的です。

## 1. 型番が国内向けか

この連載では、LN6001-JPを前提にします。

似た名前の海外向けモデルや並行輸入品ではなく、日本向けの型番を確認します。

## 2. 公式サポートに国内向け説明があるか

Linksys公式サポートで、LN6001-JPの製品情報、FAQ、ファームウェア、セットアップ手順が案内されています。

特に、次のような説明は確認しておきたいところです。

- 技適認証
- 国内法令適合
- 日本国内専用
- 電波出力や地域コード変更不可

ここが確認できると、かなり安心です。

## 3. 国内販売代理店のページがあるか

株式会社アスクの製品ページでも、Velop WRT Pro 7はOpenWrtベースのWi-Fi 7対応トライバンドルーターとして案内されています。

国内流通品として購入する場合は、販売店や代理店の情報も確認しておくと安心です。

## 4. 本体ラベルを確認できるか

実機を購入したら、本体底面のラベルを確認します。

SSID、パスワード、管理情報だけでなく、型番や認証表示も確認しておくと、あとから説明しやすくなります。

ただし、本体ラベルの写真をSNSや記事へ出さないでください。

初期SSID、Wi-Fiパスワード、管理用情報が含まれることがあります。

## 5. 海外版の仕様と混同していないか

海外版や別地域向けモデルと、日本向けLN6001-JPでは、無線仕様や最大通信速度の表記が異なる場合があります。

Linksys公式サポートでは、日本国内では5GHz帯での240MHz帯域幅の使用が制限されるため、実際の最大通信速度は9,239Mbpsになると説明されています。

「海外レビューではもっと速い数字を見た」という場合でも、日本向けモデルの仕様を基準に見ます。

ここはちょっと地味ですが、大事です。

## 技適を確認したい時

製品購入前後に「技適ってどこで確認するの？」となることがあります。

まずは、次の順番が現実的です。

```txt
製品ページを見る
  ↓
公式サポートを見る
  ↓
本体ラベルを見る
  ↓
必要なら総務省の技適検索を見る
  ↓
不安ならメーカーや販売代理店へ確認する
```

技適や認証の確認方法については、TELECのFAQでも、総務省電波利用ポータルの「技術基準適合証明等を受けた機器の検索」で調べられると案内されています。

ただし、一般ユーザーが検索結果だけで全てを判断するのは難しいこともあります。

業務利用や店舗利用で不安がある場合は、販売店、メーカー、代理店へ確認するのが安全です。

## 購入後に避けたい操作

国内向けWi-Fiルーターとして使う前提では、無線の規制領域に関わる値を不用意に触る必要はありません。

特に、次のような操作は避けます。

- 国設定の変更
- 地域コードの変更
- 送信出力の変更
- DFS回避を目的にした設定変更
- 国内向けに許可されていないチャネル利用
- 6GHz帯を海外設定に寄せる操作
- 無線ドライバや無線ファームウェアを置き換える操作

OpenWrtの情報を読んでいると、`country` や `txpower`、チャネル幅などを変更する例に出会うことがあります。

海外の検証記事としては面白いです。

でも、日本国内向けWi-Fiルーターの実用記事としては、そのまま真似しないほうがよいです。

この連載では、技適や国内利用の前提に関わる無線設定は変更しません。

## CLIで確認するなら「読み取り」だけで十分

技適や国内向け製品としての扱いは、ユーザーがSSHで何かを書き換えて証明するものではありません。

購入判断では、公式サポート、国内製品ページ、本体ラベル、製品型番の確認が中心です。

一方で、OpenWrtベース製品として「今どの状態で動いているか」を見るだけなら、CLIは役に立ちます。

まずはシステム情報です。

```sh
echo "### system"
date
ubus call system board
uptime
free
df -h

echo "### base packages"
opkg list-installed | grep -E 'base-files|busybox|dnsmasq|firewall|dropbear|uhttpd'
```

ここでは、ファームウェアのベース情報、起動時間、メモリ、ストレージ、基本パッケージを確認します。

設定は変更しません。

ネットワーク状態も読み取りだけにします。

```sh
echo "### interface status"
ifstatus wan
ifstatus wan6

echo "### routes"
ip route show
ip -6 route show

echo "### clients"
cat /tmp/dhcp.leases
```

WAN / WAN6の状態、IPv4 / IPv6の経路、LAN側端末の一覧を確認できます。

あとでIPoE、DNS、VPNの記事へ進む時の基準点になります。

Wi-Fiまわりは、特に慎重に扱います。

```sh
echo "### wireless config"
uci show wireless

echo "### radio status"
wifi status

echo "### wireless interfaces"
iw dev

echo "### regulatory logs"
logread | grep -iE 'dfs|radar|regulatory|country|txpower' | tail -50
```

これらは状態確認のためのコマンドです。

無線の国設定、送信出力、DFS関連の値を変更する目的では使いません。

## OpenWrtを否定しない。むしろ、うまく使うために線を引く

この話をすると、

```txt
結局OpenWrtは危ないの？
```

と思う人がいるかもしれません。

そうではありません。

OpenWrtは、ネットワークを理解し、用途に合わせて調整するためのとても強力な環境です。

問題は、Wi-Fiルーターの場合、自由に触れる範囲の中に「触るべきではない領域」も混ざって見えることです。

たとえば、次のような設定は家庭や小さなオフィスでもかなり役に立ちます。

- VLAN
- DHCP
- DNS
- VPN
- firewall zone
- バックアップ
- ログ確認

一方で、次のようなものは国内向けWi-Fiルーターとしての前提に関わります。

- 無線の国設定
- 地域コード
- 送信出力
- DFS
- チャネル制御

この2つを分けて考えると、OpenWrtの自由度を現実的に活かしやすくなります。

この連載では、OpenWrtらしい柔軟性は活かしつつ、国内向けWi-Fiルーターとしての前提は崩さない方針で進めます。

## ゲストWi-FiやVPNは、技適とは別レイヤーで考える

ゲストWi-Fiを作る。  
VLANでStaffとGuestを分ける。  
TailscaleやWireGuardで外出先から入る。  
DHCP予約でNASのIPを固定する。

こうした設定は、主にネットワーク設計の話です。

これは技適の話とは別レイヤーです。

技適や国内法令への適合は、

```txt
このWi-Fiルーターが日本国内で電波を出す機器としてどう扱われているか
```

の話です。

ゲストWi-Fi、VLAN、VPN、DNS、DHCP、firewallは、その上でどうネットワークを設計するかの話です。

ここを分けておくと、OpenWrt系ルーターの自由度を怖がりすぎず、かといって無線規制まわりを雑に扱わずに済みます。

## LN6001-JPを選ぶ理由を一言でいうと

LN6001-JPを選ぶ理由を一言でいうと、こうです。

```txt
OpenWrt的な柔軟性を使いたい。
でも、日本国内でWi-Fiルーターとして使う前提は崩したくない。
```

この両方を取りやすいのが、LN6001-JPの立ち位置です。

一般市販ルーターに非公式ファームウェアを入れるところから始めるのではなく、OpenWrtベースの国内向けWi-Fi 7ルーターとして使い始める。

そのうえで、必要に応じてLuCI、SSH、VLAN、VPN、IPoE / IPv6、DNSなどを少しずつ扱っていく。

家庭・小さなオフィス・店舗での実用を考えるなら、この順番が現実的です。

日本で使うなら、

```txt
OpenWrtが動くか
```

だけでなく、

```txt
国内向けWi-Fi機器としてどう扱われているか
```

## CLI例の前提

## よくある質問

### OpenWrtが動けば、日本でもそのままWi-Fiルーターとして使っていい？

技術的に動くことと、日本国内でWi-Fi機器として安心して使えることは同じではありません。

Wi-Fiルーターは電波を出す無線機器なので、国内向け製品としてどう案内されているか、技適や国内法令への適合がどう説明されているかを確認する必要があります。

この連載では、一般市販ルーターへ非公式ファームウェアを入れる手順ではなく、国内向けOpenWrtベース製品として案内されているLN6001-JPを題材にします。

### 技適の話は難しく考えないとだめ？

制度を全部読み込む必要はありません。

実用上は、まず次の点を確認するところからで十分です。

- 日本向け型番か
- 国内向け製品ページがあるか
- 公式サポートで技適・国内法令への適合が案内されているか
- 本体ラベルで型番や認証表示を確認できるか
- 無線の国設定や出力変更をユーザーが触る前提になっていないか

細かな判断が必要な場合は、公式サポートや販売代理店に確認するのが安全です。

### LN6001-JPはなぜ技適の文脈で触れやすい？

LN6001-JPは、OpenWrtベースの製品として国内向けに案内され、Linksys公式サポートでも技適認証・国内法令適合が説明されているからです。

また、日本国内専用として、無線電波出力等の仕様を一部調整したモデルであり、電波出力や地域コードの変更はできないと案内されています。

ユーザーが自分で一般市販ルーターへ非公式ファームウェアを書き込む前提ではない点が大きな違いです。

### ゲストWi-FiやVLANを設定すると技適に関係する？

通常、ゲストWi-FiやVLANはネットワーク分離の設計であり、技適そのものを変更する話ではありません。

ただし、Wi-Fiの国設定、地域コード、送信出力、DFS、チャネル制御のような無線制御に関わる項目は別です。

この連載では、ゲストWi-Fi、VLAN、VPN、DNS、DHCPなどのネットワーク運用は扱いますが、国内向け製品としての前提に関わる無線設定は変更しません。

### SSHで確認したほうがいい？

必須ではありません。

購入判断では、公式サポート、国内製品ページ、本体ラベル、型番確認のほうが重要です。

SSHは、OpenWrtベース製品としてシステム状態やネットワーク状態を確認するための補助として使います。

最初は読み取り専用コマンドだけで十分です。

### 技適はどこで確認できますか？

まずは製品ページ、公式サポート、本体ラベルを確認します。

必要に応じて、総務省の「技術基準適合証明等を受けた機器の検索」でも確認できます。

業務利用や店舗利用で不安がある場合は、メーカーや販売代理店へ確認するのが安全です。

## この連載で使っているOpenWrtルーター

この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使っています。

https://amzn.to/4tVu4yG

LN6001-JPは、国内向けに展開されているOpenWrtベースのWi‑Fi 7ルーターです。一般的な「自分でOpenWrtを書き込むタイプ」と違い、OpenWrtベースで最適化されたファームウェアを搭載した状態で出荷されるため、かなり導入しやすいのが特徴です。

初期セットアップ済みで、基本的には電源を入れてすぐ利用可能。LuCIブラウザ管理画面、SSH、opkg、VLAN、OpenVPN、WireGuard、Tailscaleなど、OpenWrtらしい拡張性も備えています。

特に日本の回線環境向けとして、Linksys公式の「オートIPoE」モジュールが用意されているのが大きなポイントです。OCNバーチャルコネクト、transixなどのIPv4 over IPv6環境でも、LuCIから比較的シンプルに設定できます。

もちろんCLIで細かく調整していく楽しさもありますが、MAP-EやIPIPは設定項目がかなり多いため、「まず普通に安定して使いたい」という場合はオートIPoEを使った方がかなり楽です。

この連載では、実機開発にも関わった立場から、家庭・小規模オフィス・店舗などで“実際どう使うと快適なのか” を中心に、現実的なOpenWrt運用を紹介していきます。
