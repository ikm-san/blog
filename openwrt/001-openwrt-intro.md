# OpenWrtルーターって何が違うの？普通のWi-Fiルーターで物足りなくなった人へ【OpenWrt集中連載001】

普通のWi-Fiルーターって、かなり優秀です。

箱から出して、回線につないで、スマホでSSIDを選んで、パスワードを入れる。  
多くの家庭では、それだけで普通にインターネットが使えます。

これは本当にすごいことです。

でも、少し使い込んでくると、ある日こう思うことがあります。

「ゲストWi-Fiって、本当に家庭内LANから分かれてる？」  
「子ども用端末だけDNSを変えたい」  
「スマート家電を親のPCやNASと同じ場所に置きたくない」  
「外出先からNASや店舗の録画機へ入りたい」  
「今、何がつながっていて、どこで詰まっているのか見たい」

ここで、普通のWi-Fiルーターだと急に見える範囲が狭くなります。

設定画面にはボタンがある。  
でも、その先を細かく見られない。  
ログも少ない。  
firewallの中身も分からない。  
VLANやVPNも、メーカーが用意した範囲まで。

そこで出てくるのがOpenWrt系ルーターです。

OpenWrt系ルーターは、ざっくり言うと **ネットワークを自分で見て、分けて、育てられるルーター** です。

ただし、ここで身構えなくて大丈夫です。

OpenWrtと聞くと、

「CLIで全部設定するのでは？」  
「黒い画面でコマンドを間違えたら壊れそう」  
「日本の回線で本当に普通に使えるの？」  
「技適とか大丈夫なの？」

と不安になる人も多いと思います。

この連載では、その不安をひとつずつほどいていきます。

題材にするのは、OpenWrtベースのWi-Fi 7ルーター **Linksys Velop WRT Pro 7（LN6001-JP）** です。

LN6001-JPは、OpenWrtベースの自由度がありながら、国内向けモデルとして初期設定済みで出荷されます。LuCI、SSH、opkg、VLAN、VPN、IPoEなどへ進めますが、最初から全部を触る必要はありません。

まず普通に使う。  
必要になったら少し足す。  
触る前にバックアップを取る。  
困ったらログを見る。

このくらいの距離感で始めるのが、OpenWrtベースルーターとは長く付き合いやすいと思っています。

> この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使いながら、家庭・小さなオフィス・店舗でOpenWrtベースルーターを現実的に使う方法をまとめています。

## 先にざっくり結論

OpenWrt系ルーターの違いをひとことで言うと、これです。

```txt
ネットワークを見える化して、分けて、必要な機能をあとから足せる
```

普通のWi-Fiルーターは、かんたんに使える“家電”として優秀です。

一方でOpenWrt系ルーターは、家庭や店舗のネットワークをもう少し自分の用途に合わせて整理できます。

たとえば、こんなことがしやすくなります。

- ゲストWi-Fiを家庭内LANからちゃんと分ける
- Kids / IoT / Staff / Guestのように用途別ネットワークを作る
- NASやプリンターのIPを固定する
- TailscaleやWireGuardで外出先から安全に入る
- DNS広告ブロックやFamily DNSを使う
- IPoE / IPv6 / WANの状態を確認する
- ログを見て、つながらない原因を切り分ける
- 設定をバックアップして戻せるようにする

ただし、最初から全部を触る必要はありません。

むしろおすすめは逆です。

```txt
まず普通に使う
  ↓
困ったところだけ分ける
  ↓
必要な機能だけ足す
  ↓
変更前にバックアップする
```

これくらいがちょうどいいです。

## OpenWrtって何？

OpenWrtは、Wi-Fiルーターや組み込み機器向けに使われているLinuxベースのルーターOSです。

ざっくり言うと、普通のルーターでは隠れている部分を、もう少し自分で見たり触ったりできるようにする世界です。

OpenWrtのルーツとしてよく語られるのが、Linksys WRT54Gです。

2000年代前半に人気を集めたこの製品をきっかけに、「家庭用ルーターをもっと自由に使いたい」という流れが広がり、OpenWrtプロジェクトへ発展していきました。

普通のWi-Fiルーターでは、メーカーが用意した機能を使うことが前提です。

OpenWrtでは、追加パッケージを入れて機能を増やしたり、LuCIというWeb管理画面や設定ファイルを通じて、ネットワーク構成を細かく調整したりできます。

できることの例は、こんな感じです。

- ゲストWi-Fiを家庭内LANから分離する
- 子ども用端末だけDNS設定を変える
- IoT機器をMain LANから分ける
- VPNで自宅や小さなオフィスへ安全に入る
- VLANでStaff、Guest、Deviceを分ける
- DNS広告ブロックを追加する
- ログや接続端末を見ながらトラブルを切り分ける
- 設定をバックアップして、戻せる状態を作る

つまりOpenWrtは、単なる「Wi-Fiを飛ばす箱」ではなく、ネットワークを自分の用途に合わせて育てられる環境です。

## 普通のWi-Fiルーターは悪くない

ここは最初に言っておきたいです。

普通の家庭用Wi-Fiルーターは、悪いものではありません。

むしろ、多くの家庭ではかなり合理的です。

- 買ってすぐ使える
- スマホアプリで設定しやすい
- 設定項目が少なく、迷いにくい
- サポートが受けやすい
- 家族にも説明しやすい

この「迷わず使える」ことは、めちゃくちゃ大事です。

家庭内ネットワークは、使っているのが自分だけとは限りません。

家族がいます。  
子どもがいます。  
テレビがあります。  
ゲーム機があります。  
プリンターが急に使えないと怒られます。

なので、普通のルーターが“かんたん”を重視するのは正しいです。

ただ、最近は家庭の中にネットワーク機器が増えすぎました。

スマートフォン、PC、ゲーム機、テレビ、NAS、プリンター、監視カメラ、スマート家電、スマートスピーカー、ロボット掃除機。

さらに店舗なら、POS、決済端末、予約端末、録画機、ゲストWi-Fi。

こうなると、

```txt
全部同じWi-Fiでいいの？
```

という問題が出てきます。

ここでOpenWrt系ルーターの出番です。

## OpenWrtで変わること

OpenWrt系ルーターを使うと、普通のWi-Fiルーターでは見えにくいところを確認しながら、少しずつ整理できるようになります。

### 1. ネットワークを分けやすい

家庭なら、Main、Guest、Kids、IoT。

店舗なら、Staff、Guest、Device、Camera。

小さなオフィスなら、社内端末、ゲストWi-Fi、検証用ネットワーク。

こうした分け方を作りやすいのが、OpenWrt系ルーターの大きな魅力です。

ここで大事なのは、SSIDを増やすこと自体ではありません。

大事なのは、

```txt
通信してよいもの
分けたいもの
外から入ってよいもの
絶対に見せたくないもの
```

を整理できることです。

ゲストWi-Fiはインターネットには出ていい。  
でもNASやプリンターのある部屋には入れない。

firewall zoneは、最初はこの「部屋分け」くらいで考えると分かりやすいです。

### 2. 必要な機能をあとから足せる

OpenWrtでは、パッケージ管理の仕組みを使って機能を追加できます。

たとえば、次のようなものです。

- 通信確認用のツール
- DNS広告ブロック
- DDNS
- VPN
- ログ確認
- firewall補助機能
- LuCIの追加画面

普通のWi-Fiルーターでは、メーカーが用意していない機能は基本的に使えません。

OpenWrt系ルーターでは、必要になった機能をあとから足せる余地があります。

ただし、ここで調子に乗って何でも入れるのはおすすめしません。

ルーターはPCではありません。  
ストレージもメモリも限られています。

`opkg upgrade` で全部更新、みたいなノリも基本的には避けます。

必要なものだけ入れる。  
入れる前にバックアップする。  
これが安全です。

### 3. 状態を確認しやすい

ネットワークのトラブルで一番困るのは、

```txt
何が起きているのか分からない
```

ことです。

OpenWrt系ルーターでは、LuCIやSSHで次のような状態を確認しやすくなります。

- どの端末が接続しているか
- 端末にどのIPアドレスが配られているか
- DNSがどこを向いているか
- WAN側がどうつながっているか
- IPv6が生きているか
- firewallやVPNが期待通り動いているか
- ログに何が出ているか

最初は設定を書き換える必要はありません。

見るだけでかなり価値があります。

たとえばSSHで見るなら、最初はこのくらいです。

```sh
ifstatus wan
ifstatus wan6
cat /tmp/dhcp.leases
logread | tail -n 50
```

これだけでも、WAN、IPv6、DHCP、ログが見えます。

黒い画面は怖そうですが、最初は読み取りだけでOKです。

### 4. バックアップして戻しやすい

OpenWrt系ルーターで一番大事なのは、設定を全部暗記することではありません。

変更前に戻れる状態を作っておくことです。

LuCIからバックアップを取れます。

```txt
System → Backup / Flash Firmware → Generate archive
```

SSHなら、主要設定ファイルを保存できます。

```sh
uci show network > network-before.txt
uci show wireless > wireless-before.txt
uci show firewall > firewall-before.txt
uci show dhcp > dhcp-before.txt
```

バックアップが1本あるだけで、VLANもfirewallもVPNも、急に怖くなくなります。

逆に、バックアップなしで深い設定へ進むと、ちょっとしたミスで迷子になります。

OpenWrt系ルーターでは、

```txt
触る前にバックアップ
動いたらまたバックアップ
```

これがかなり大事です。

## でも、最初からCLIで全部やらなくていい

OpenWrtと聞くと、黒い画面でコマンドを打ち続けるイメージがあるかもしれません。

でも、LN6001-JPにはLuCIというWeb管理画面があります。

Wi-Fi設定、ネットワーク設定、DHCP、firewall、ソフトウェア管理など、多くの操作はブラウザから確認・変更できます。

SSHやCLIは、最初から設定を書き換えるためではなく、まずは確認用として使うのがおすすめです。

たとえば、

- 現在のIPアドレスを見る
- 接続中端末を見る
- ログを見る
- 設定内容を読む
- 変更前の状態を保存する

このくらいで十分です。

最初は、

```txt
設定変更はLuCI
状態確認はSSH
```

くらいの使い分けがちょうどいいと思います。

## 日本では「市販ルーターにOpenWrtを入れればOK」とは言いにくい

OpenWrtについて調べると、市販Wi-FiルーターにOpenWrtをインストールする手順がたくさん見つかります。

対応機種を探して、ファームウェアを書き込んで、自分好みのルーターにする。  
これはOpenWrt文化としてはとても面白い世界です。

ただし、日本国内でWi-Fi機器を使う場合は、技適や国内法令の前提を無視できません。

Wi-Fiルーターは、電波を出す無線機器です。

一般的な市販Wi-Fiルーターへ非公式ファームウェアを書き込んだ場合、次のような無線制御が国内向け製品として適切に保たれているかを、利用者側で保証するのは簡単ではありません。

- 周波数
- 送信出力
- チャネル
- DFS
- 国設定

海外の記事で普通に紹介されている手法でも、日本国内でそのまま参考にするのは注意が必要です。

少なくとも、家庭や小さなオフィス、店舗で安定運用したい人に対して、

```txt
市販ルーターに非公式OpenWrtを入れれば大丈夫
```

とは言いにくいです。

そのため、この連載では、一般的な市販Wi-Fiルーターへ非公式ファームウェアを書き込む手順は扱いません。

## LN6001-JPを題材にする理由

この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を題材にします。

理由はシンプルです。

LN6001-JPは、OpenWrtベースのWi-Fi 7ルーターとして国内向けに展開されており、OSインストールや初期設定を済ませた状態で使い始められるからです。

通常、OpenWrtを一から導入する場合は、次のような作業が必要になります。

- 対応機種を調べる
- ファームウェアを入手する
- 書き換え手順を確認する
- 失敗時の復旧方法を調べる
- 初期設定を自分で組み立てる

これはこれで楽しいです。

でも、家庭や仕事で使うルーターとしては、入口の負担が少し重いです。

実際、検証中に何台かソフトブリック化したこともあります。  
あれはあれで学びはありますが、日常で使うルーターではあまり味わいたくないイベントです。

その点、LN6001-JPは「OpenWrtを導入するところ」ではなく、「OpenWrtベースの環境をどう使うか」から始められます。

Linksys公式サポートでも、LN6001-JPは、Wi-Fi 7トライバンド、LuCI、SSH、opkg、スクリプト設定に対応したOpenWrtベースのWi-Fi 7ルーターとして案内されています。

さらに、NTTフレッツ系のIPoE / IPv4 over IPv6についても、追加モジュール「オートIPoE」による対応が案内されています。

日本の回線環境で使ううえで、ここはかなり大事です。

## LN6001-JPでできることのイメージ

LN6001-JPは、単なる「速いWi-Fi 7ルーター」として見るだけでは少しもったいない製品です。

OpenWrtベースの柔軟性があるので、次のような使い方へ広げやすくなります。

- LuCIでWeb管理する
- SSHで状態やログを確認する
- opkgで必要なパッケージを追加する
- ゲストWi-FiをLANから分ける
- VLANでStaff / Guest / Deviceを分ける
- WireGuardやTailscaleでリモートアクセスする
- オートIPoEモジュールで日本のIPv4 over IPv6環境に対応する
- 設定をバックアップして復元できるようにする

ハードウェア面でも、2.4GHz / 5GHz / 6GHz のトライバンドWi-Fi 7、2.5Gbps WAN、1Gbps LANポート4基、1GB DDR4メモリなど、家庭だけでなく小さなオフィスやSOHOでも使いやすい構成です。

もちろん、全部を一気に使う必要はありません。

最初は普通のWi-Fiルーターとして使う。  
必要になったらゲストWi-Fiを分ける。  
そのあとDNS、VPN、VLANへ進む。

この順番が現実的です。

## 家庭でOpenWrt系ルーターを使う意味

家庭でOpenWrt系ルーターを使う意味は、難しい設定を全部使いこなすことではありません。

むしろ、次のようなことを少しずつできるようになることです。

- 家族用とゲストWi-Fiを分ける
- IoT機器を家庭内のメイン端末から分ける
- 子ども用端末のDNSや利用ルールを考える
- NASやプリンターの接続状況を把握する
- 外出先から安全に自宅へアクセスする
- トラブル時に、どこで止まっているか切り分ける

家庭ネットワークは、完璧な設計より「家族に説明できる設計」のほうが強いです。

```txt
普段はHomeWifi
ゲストにはHomeWifi_Guest
スマート家電はHomeWifi_IoT
```

最初はこのくらいで十分です。

OpenWrt系ルーターは、そこから必要になった分だけ育てられます。

## 小さなオフィスや店舗で使う意味

小さなオフィスや店舗では、家庭よりも「分けること」の重要度が上がります。

たとえば店舗なら、Staff、Guest、POS、Camera。

小さなオフィスなら、社内PC、ゲストWi-Fi、検証端末、業務機器。

これらを同じWi-Fiや同じLANにまとめてしまうと、あとから管理しにくくなります。

OpenWrt系ルーターでは、SSID、VLAN、firewall zone、VPNなどを組み合わせて、必要な通信だけを許可する構成を考えやすくなります。

外出先から拠点へ入りたい場合も、ポート開放だけに頼るのではなく、WireGuardやTailscaleのようなVPNを検討しやすくなります。

小さな環境だからこそ、最初に軽く整理しておく。

これだけで、あとからかなり助かります。

## 最初に触るならこの範囲で十分

OpenWrtベースのルーターだからといって、最初からすべての設定項目を理解する必要はありません。

まずはこのくらいで十分です。

- LuCI管理画面へログインする
- 現在の回線方式とWAN状態を確認する
- Wi-Fi名とパスワードを確認・変更する
- LAN側IPアドレスを確認する
- 設定バックアップの場所を確認する
- SSHでログインできるか確認する

最初からVLAN、VPN、パッケージ追加まで一気に触ると、どこで問題が起きたのか分かりにくくなります。

ネットワーク設定は、

```txt
一つ変更する
動作確認する
メモする
バックアップする
```

の繰り返しが大事です。

特に次の設定を触る前には、必ずバックアップを取るのがおすすめです。

- LAN IPアドレス
- DHCP
- VLAN
- firewall
- VPN
- Wi-FiのSSIDや暗号化設定

OpenWrtは自由度が高いです。

だからこそ、戻せる状態を作ってから触るのが基本です。

## OpenWrtが向いている人、向いていない人

OpenWrt系ルーターは、「全部おまかせの家電」として使いたい人よりも、自分の用途に合わせて少しずつ調整したい人に向いています。

向いているのは、たとえば次のような人です。

- 市販ルーターの簡易設定では物足りない
- ゲストWi-FiやIoT機器をきちんと分けたい
- VPNやVLANを使ってみたい
- 家庭内や店舗内のネットワークを見える化したい
- 設定内容を理解しながら運用したい
- 小さなオフィスやSOHOのネットワークを整理したい
- 変更前にバックアップを取る運用ができる

一方で、次のような人には少し重く感じるかもしれません。

- スマホアプリだけで全部自動設定したい
- ネットワークの仕組み自体を見たくない
- 設定項目が多いだけで負担に感じる
- トラブル時にログや状態を確認したくない

OpenWrtの魅力は、できることが多いことです。

でも、その自由度の高さが、最初は「情報量が多すぎて、おえっとなる」原因にもなります。

なので、最初は本当に小さく始めるくらいで大丈夫です。

## CLI例の前提について

これらは国内法令や技適に関わる部分でもあるため、「OpenWrtだから何でも自由に変更する」のではなく、国内向け製品として必要な制御を維持したうえで、ネットワーク部分を柔軟に扱う考え方で進めます。

## よくある質問

### OpenWrtは普通の家庭用ルーターと何が違う？

普通の家庭用Wi-Fiルーターは、メーカーが用意した機能の範囲で簡単に使うことを重視しています。

OpenWrt系ルーターは、Wi-Fi、DNS、DHCP、VLAN、VPN、firewallなどを、自分の環境に合わせて細かく設計・調整しやすいのが違いです。

ただし、そのぶん設定項目は増えます。

最初から全部を使おうとせず、まずはLuCIで状態を見て、必要になった機能だけ少しずつ足していくのがおすすめです。

### OpenWrt初心者でも使える？

使えます。

ただし、最初からすべてを理解しようとすると大変です。

まずはLuCIのWeb管理画面で、回線状態、Wi-Fi設定、接続端末、バックアップの場所を確認するところから始めるのがよいと思います。

SSHを使う場合も、最初は設定を書き換えるより、ログや現在の設定を見る用途から始めると安全です。

### 日本で市販Wi-FiルーターにOpenWrtを入れてもいい？

この連載では、一般的な市販Wi-Fiルーターへ非公式にOpenWrtを書き込んで使う方法は扱いません。

日本国内でWi-Fi機器を使う場合は、技適や国内法令の前提が重要です。

非公式ファームウェアへ書き換えた場合、周波数、送信出力、DFS、チャネル制御などが国内向け製品として適切な状態に保たれているかを、利用者側で保証するのは簡単ではありません。

そのため、この連載では国内向けに展開されているOpenWrtベース製品であるLN6001-JPを題材にします。

### LN6001-JPは全部細かく設定しないと使えない？

いいえ。

LN6001-JPは工場出荷時点で基本設定が済んでいるため、DHCP自動で利用できる一般的な環境であれば、まず普通のWi-Fiルーターとして使い始められます。

そのうえで、必要に応じてLuCI、SSH、VLAN、VPN、IPoE/IPv6などへ踏み込める、という立ち位置です。

最初から全部を設定するより、まずは安定して使える状態を作り、必要な機能をあとから足していく進め方がおすすめです。

### SSHやCLIは必須？

必須ではありません。

LuCIだけでもかなり操作できます。

ただ、トラブル時にSSHで次のような読み取りコマンドを使えると、かなり楽になります。

```sh
ifstatus wan
ifstatus wan6
cat /tmp/dhcp.leases
logread | tail -n 50
```

最初は見るだけでOKです。

### OpenWrtは家庭にはやりすぎ？

人によります。

ただ、最近の家庭はネットワーク機器が多いです。

スマートフォン、PC、NAS、プリンター、テレビ、ゲーム機、スマート家電、見守りカメラなどが増えると、「全部同じLANでいいのか？」という場面が出てきます。

ゲストWi-FiやIoT分離だけでも、OpenWrt系ルーターを使う意味はあります。

### まず何から触ればいい？

まずはこの順番がおすすめです。

1. LuCIへログインする
2. WAN / WAN6の状態を見る
3. Wi-Fi名とパスワードを確認する
4. 設定バックアップを取る
5. 必要ならゲストWi-Fiを作る

最初からVLANやVPNへ行かなくて大丈夫です。

## この連載で使っているOpenWrtルーター

この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使っています。

https://amzn.to/4tVu4yG

LN6001-JPは、国内向けに展開されているOpenWrtベースのWi‑Fi 7ルーターです。一般的な「自分でOpenWrtを書き込むタイプ」と違い、OpenWrtベースで最適化されたファームウェアを搭載した状態で出荷されるため、かなり導入しやすいのが特徴です。

初期セットアップ済みで、基本的には電源を入れてすぐ利用可能。LuCIブラウザ管理画面、SSH、opkg、VLAN、OpenVPN、WireGuard、Tailscaleなど、OpenWrtらしい拡張性も備えています。

もちろんCLIで細かく調整していく楽しさもありますが、MAP-EやIPIPは設定項目がかなり多いため、「まず普通に安定して使いたい」という場合はオートIPoEを使った方がかなり楽です。
