# LN6001-JPってどんなルーター？｜普通のWi-Fi 7ルーターとOpenWrt自作機のあいだにある選択肢【OpenWrt集中連載003】

Wi-Fi 7ルーターを探していると、だいたい同じ言葉が並びます。

速い。  
広い。  
メッシュ。  
アプリで簡単。  
家じゅう快適。

もちろん、それは大事です。

Wi-Fiルーターなので、速くて安定していることは大前提です。  
家族から「Wi-Fi遅いんだけど」と言われ続けるルーターは、どれだけ高機能でもつらいです。

ただ、Linksys Velop WRT Pro 7（LN6001-JP）は、そこだけを見ると少しもったいない製品です。

このルーターの面白さは、

```txt
Wi-Fi 7のハードウェア
+
OpenWrtベースの自由度
+
日本向け製品としての始めやすさ
```

が同じ箱に入っているところです。

普通のWi-Fi 7ルーターとして使い始められる。  
でも、必要になったらLuCIで細かく見られる。  
SSHでログを読める。  
opkgで機能を足せる。  
VLANやVPN、IPoE/IPv6まわりにも踏み込める。

つまり、LN6001-JPは、

```txt
全部おまかせの家電ルーター
```

と、

```txt
自分でOpenWrtを書き込んで作るルーター
```

のちょうど中間にいるような製品です。

この記事では、LN6001-JPがどんな立ち位置のルーターなのかを、普通の市販Wi-Fiルーター、自作OpenWrtルーター、日本の回線事情という3つの視点から見ていきます。

買う前に「自分に合いそうか」を判断するための記事です。

> この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使いながら、家庭・小さなオフィス・店舗でOpenWrtベースルーターを現実的に使う方法をまとめています。  
> 連載目次: [LN6001-JPで始めるOpenWrtベースルーター実践ガイド](https://note.com/ikmsan/n/ndf7569fea475)

- LN6001-JPがどんな人に向いているか分かる
- 逆に、どんな人には少し濃すぎるか分かる
- 買う前に確認したいポイントが分かる

ざっくり言うと、

```txt
LN6001-JPは「速いWi-Fiルーター」だけではなく、
あとからネットワークを育てられるルーター
```

です。

## 先にざっくり結論

LN6001-JPの魅力は、この3つを同時に取りやすいところです。

1. Wi-Fi 7世代のハードウェア
2. OpenWrtベースの柔軟性
3. 日本国内向け製品としての始めやすさ

普通の市販ルーターのように、まずWi-Fiルーターとして使い始められます。

でも、必要になればLuCIで設定を見られます。  
SSHで状態を読めます。  
opkgで機能を追加できます。  
VLAN、VPN、IPoE/IPv6まわりにも進めます。

ただし、

```txt
全部アプリで自動がいい
ネットワークの中身は一切見たくない
```

という人には少し向きません。

LN6001-JPは、ネットワークを少しずつ理解しながら、自分の家庭・小さなオフィス・店舗に合わせて整えていきたい人に向いた製品です。

## こういう人向けです

LN6001-JPは、こんな人に刺さりやすいです。

- Wi-Fi 7ルーターに買い替えたい
- でも、ただ速いだけでは少し物足りない
- ゲストWi-Fiを家庭内LANからちゃんと分けたい
- NAS、ミニPC、ホームラボを使っている
- TailscaleやWireGuardで外から入りたい
- 店舗や小さなオフィスでStaff / Guest / Deviceを分けたい
- IPoEやIPv6の状態を自分でも確認したい
- LuCIやSSHでログや設定を見られると安心する
- OpenWrtに興味はあるけど、自分でファームウェアを書き込むのは少し重い

逆に、次のような人には少し濃いかもしれません。

- スマホアプリだけで全部終わらせたい
- 設定画面をほとんど見たくない
- ゲストWi-Fi以上のことはしない
- VLAN、VPN、DNS、IPoEに興味がない
- ルーターにSSHで入ること自体に強い抵抗がある
- とにかく安くて簡単なWi-Fi 7ルーターだけが欲しい

LN6001-JPは、ただ置いて終わりのルーターというより、

```txt
最初は普通に使って、
必要になったら少しずつ育てるルーター
```

として見ると分かりやすいです。

## LN6001-JPを3つの軸で見る

LN6001-JPは、次の3つの軸で見ると理解しやすいです。

![表画像 table-01](https://raw.githubusercontent.com/ikm-san/blog/main/openwrt/assets/003/table-01.png)

この3つのうち、どれか1つだけで見ると少しズレます。

単なるWi-Fi 7ルーターとして見ると、OpenWrtベースの自由度を見落とします。

OpenWrtルーターとしてだけ見ると、工場出荷時に初期設定された国内向け製品としての始めやすさを見落とします。

国内向け製品としてだけ見ると、普通の簡単ルーターより深く触れる面白さを見落とします。

LN6001-JPの価値は、この3つが重なっているところにあります。

## 普通のWi-Fiルーターと何が違うのか

一般的な家庭用Wi-Fiルーターは、迷わず使えるように作られています。

これは本当に大事です。

回線につなぐ。  
スマホでSSIDを選ぶ。  
パスワードを入れる。  
あとは家族が普通に使える。

多くの家庭では、これで十分です。

普通のWi-Fiルーターは悪くありません。  
むしろ、かなりよくできています。

ただ、使い込んでいくと、こういう欲が出てきます。

```txt
ゲストWi-Fiを家庭内LANから分けたい
IoT機器だけ別ネットワークにしたい
NASやプリンターのIPを固定したい
外出先からVPNで入りたい
DNS広告ブロックを試したい
IPv6やIPoEの状態を見たい
つながらない時にログを見たい
```

ここで、普通のWi-Fiルーターだと急に見える範囲が狭くなることがあります。

メニューはある。  
でも深くは見られない。  
ログも少ない。  
firewallの中身も見えない。  
VPNやVLANもメーカーが用意した範囲まで。

LN6001-JPは、ここをもう少し自分で見られます。

もちろん、最初からCLIで全部設定する必要はありません。

まずLuCIで見る。  
必要ならSSHでログを見る。  
さらに必要ならVLANやVPNへ進む。

この段階的な進み方ができます。

## 自分でOpenWrtを入れるルーターと何が違うのか

OpenWrtに詳しい人なら、

```txt
対応ルーターを買って、自分でOpenWrtを入れればいいのでは？
```

と思うかもしれません。

それはそれで、かなり楽しい世界です。

対応機種を調べる。  
ファームウェアを選ぶ。  
書き換える。  
復旧方法も調べる。  
動いた時にちょっと嬉しい。

ホームラボや検証が好きな人には、そこも含めて遊びです。

ただ、家庭・小さなオフィス・店舗で普段使いするWi-Fiルーターとして考えると、入口の負担は少し重くなります。

自作寄りのOpenWrt導入では、少なくとも次を考える必要があります。

- 対応機種かどうか
- どのファームウェアを使うか
- 書き換え手順は安全か
- 失敗した時に復旧できるか
- Wi-FiやLANが期待通り動くか
- 日本国内で無線機器としてどう扱うか

LN6001-JPは、この入口がかなり違います。

ユーザーが自分でOpenWrtをインストールする製品ではありません。

OpenWrtベースの環境が製品として用意されています。

つまり、最初にやることは、

```txt
OpenWrtを入れる
```

ではなく、

```txt
OpenWrtベースのルーターをどう使うか
```

です。

ここはかなり大きいです。

正直、普段使いのルーターで毎回「ファームウェアを書き換えて、もし失敗したら復旧して……」から始まるのは、楽しい人には楽しいですが、家族や店舗にはあまり優しくありません。

LN6001-JPは、その入口をかなり下げてくれます。

## LN6001-JPの立ち位置を一言でいうと

一言でいうと、LN6001-JPはこういう製品です。

```txt
OpenWrtの自由度を、製品として始めやすくしたWi-Fi 7ルーター
```

普通のルーターより深く触れる。

でも、自分でファームウェアを書き込むところから始める必要はない。

この“あいだ”の立ち位置が、家庭・小さなオフィス・店舗ではかなり使いやすいです。

## 買う前に見ておきたい5つのポイント

LN6001-JPを検討するなら、スペック表だけでなく、次の5つを見ておくと判断しやすいです。

## 1. 初期設定済みで使い始められるか

LN6001-JPは、OpenWrtベースのOSや初期設定が用意された状態で出荷される製品です。

公式サポートでも、DHCP自動で利用できる環境であれば、通電して回線機器につなぐだけでインターネットとWi-Fiを利用できると案内されています。

ここは、かなり大事です。

OpenWrtに興味はある。  
でも、最初からOSインストールやファームウェア書き換えは避けたい。

そういう人にとって、この入口の低さは大きなメリットです。

## 2. 日本のIPoE / IPv4 over IPv6に対応できるか

日本の家庭回線では、IPoE / IPv4 over IPv6まわりがかなり重要です。

OCNバーチャルコネクト。  
transix。  
v6プラス。  
クロスパス。  
MAP-E。  
DS-Lite。  
IPIP。

名前だけでもうお腹いっぱいになりがちです。

海外のOpenWrt情報だけでは、このあたりの日本向け回線事情は埋まりにくいです。

LN6001-JPでは、Linksys公式サポートで「オートIPoE」モジュールが案内されており、MAP-E / DS-Lite / IPIP方式のIPv4 over IPv6接続に対応できる構成が説明されています。

ここは国内で使うルーターとして、かなり大事なポイントです。

IPoEまわりは、動けば普段は意識しません。  
でも詰まると急に難しいです。

だからこそ、公式導線があるのは安心です。

## 3. LuCI、SSH、opkgを使えるか

LN6001-JPは、LuCIブラウザ管理画面、SSHログイン、opkgパッケージ追加に対応するOpenWrtベースのルーターです。

ここが普通のルーターとの大きな違いです。

LuCIがあるので、すべてをコマンドで操作する必要はありません。

でも、SSHでログインすれば、ログや状態を確認しやすくなります。

opkgを使えば、必要な機能をあとから追加できる余地もあります。

最初は、設定変更ではなく読み取りだけで十分です。

```sh
echo "### system"
ubus call system board
uptime
free
df -h

echo "### network"
ifstatus wan
ifstatus wan6

echo "### clients"
cat /tmp/dhcp.leases
```

この段階では何も変更していません。

ファームウェア情報、稼働時間、メモリ、WAN/WAN6の状態、LAN側端末を見ているだけです。

黒い画面は怖そうですが、読むだけならかなり安全です。

## 4. VLANやVPNへ進めるか

家庭でも店舗でも、ネットワークを分けたい場面は増えています。

家庭なら、

```txt
Main
Guest
Kids
IoT
```

店舗なら、

```txt
Staff
Guest
Device
POS
Camera
```

小さなオフィスなら、

```txt
社内端末
ゲストWi-Fi
検証用ネットワーク
VPN
```

こうした分離を考えると、SSID、VLAN、firewall zone、VPNの考え方が効いてきます。

LN6001-JPはVLANに対応しており、追加モジュールでWireGuard / TailscaleやIPoE系にも進めます。

もちろん、最初から全部を設定する必要はありません。

ただ、

```txt
あとから分けられる
あとからVPNを足せる
あとからログを見られる
```

という余地があるのはかなり大きいです。

## 5. Wi-Fi 7世代のハードウェアか

LN6001-JPは、2.4GHz / 5GHz / 6GHzのトライバンドに対応したWi-Fi 7ルーターです。

公式情報では、2.5Gbps WANポート、1Gbps LANポート4基、Qualcomm IPQ9554 1.5GHzクアッドコアCPU、1GB DDR4メモリなどが案内されています。

ただし、Wi-Fiルーターは最大速度だけで選ぶものではありません。

見るべきなのは、自分の環境で意味があるかです。

- Wi-Fi 7対応端末があるか
- 6GHz対応端末があるか
- 2.5GbE WANを活かせる回線か
- LAN側は1GbEで問題ないか
- ゲストWi-Fi、VPN、VLANなどを使う予定があるか
- ただ速いだけでなく、状態を見ながら運用したいか

Wi-Fi 7だけが目的なら、もっとシンプルな製品も候補になります。

Wi-Fi 7に加えて、OpenWrtベースの自由度も欲しいなら、LN6001-JPの立ち位置が見えてきます。

## ここで勘違いしやすいところ

## 「OpenWrtベースだから、全部CLIで触る製品」ではない

これはよくある誤解です。

LN6001-JPにはLuCIがあります。

つまり、Webブラウザからかなりの設定を見たり変更したりできます。

SSHやCLIは、最初は状態確認に使うだけで十分です。

```txt
設定変更はLuCI
状態確認はSSH
```

このくらいの使い分けで大丈夫です。

## 「Wi-Fi 7なら誰でも速くなる」わけではない

Wi-Fi 7の恩恵を受けるには、端末側も対応している必要があります。

6GHzやMLOも、対応端末があってこそ活きます。

古いスマホやPC、IoT機器が中心なら、まずは5GHz / 2.4GHzの安定運用が大事です。

## 「OpenWrtだから何でも自由に変えてよい」わけではない

OpenWrtは自由度が高いです。

でも、日本国内で使うWi-Fiルーターとして、無線の国設定、送信出力、DFS関連の値は触るべきではありません。

この連載でも、そこは変更しません。

自由度を使うのは、VLAN、VPN、DNS、DHCP、firewall、ログ、バックアップといったネットワーク運用の部分です。

## 家庭で使うなら

家庭でLN6001-JPを使うなら、最初から凝りすぎなくて大丈夫です。

おすすめはこの順番です。

```txt
Main Wi-Fiを安定させる
  ↓
ゲストWi-Fiを作る
  ↓
必要ならKidsやIoTを足す
  ↓
NASやプリンターをDHCP予約する
  ↓
必要ならTailscaleやAdblockへ進む
```

家庭ネットワークは、完璧な設計より「家族に説明できる設計」のほうが強いです。

たとえば、

```txt
普段は HomeWifi
ゲストには HomeWifi_Guest
スマート家電は HomeWifi_IoT
```

このくらいで十分です。

いきなりVLANやfirewallを完璧に作る必要はありません。

## 小さなオフィスで使うなら

小さなオフィスでは、家庭より少しだけ整理が必要です。

まず考えたいのは、このあたりです。

- 社内PC
- NAS
- プリンター
- ゲストWi-Fi
- VPN
- 固定IP / DHCP予約
- IPoE / PPPoE / 固定IP回線

小さなオフィスでは、

```txt
社内端末とゲストWi-Fiを混ぜない
NASやプリンターのIPを固定する
外出先から入るならVPNを使う
```

この3つだけでもかなり運用しやすくなります。

## 店舗で使うなら

店舗で使うなら、Staff / Guest / Deviceを分ける考え方がかなり効きます。

```txt
Staff:
  スタッフPC、管理端末、業務タブレット

Guest:
  ゲストWi-Fi

Device:
  カメラ、録画機、スマートロック、設備機器

POS / Payment:
  ベンダー要件を優先
```

店舗では、ゲストWi-Fiと業務端末が同じLANにいる状態は避けたいです。

ただし、POSや決済端末は雑にDeviceへ入れず、ベンダー要件を確認します。

ここはかなり現実的に大事です。

## 買う前に自分へ聞きたい質問

LN6001-JPを買う前に、次を自分に聞いてみると判断しやすいです。

```txt
Wi-Fi 7だけでなく、ネットワークを整理したい？
ゲストWi-Fiをちゃんと分けたい？
NASやミニPCを使っている？
TailscaleやWireGuardに興味がある？
IPoEやIPv6の状態を自分でも確認したい？
LuCIやSSHを少し触ってもよい？
変更前にバックアップを取る運用ができそう？
```

このうちいくつかに「はい」があるなら、LN6001-JPはかなり相性がよいと思います。

逆に、

```txt
速ければいい
設定画面は見たくない
アプリだけで全部やりたい
```

なら、もっとシンプルなWi-Fi 7ルーターでもよいかもしれません。

## 買ったあとに最初に見るところ

購入後は、いきなり高度な設定へ進むより、まず次を確認するのがおすすめです。

- 本体底面ラベルのSSID、パスワード、型番
- LuCI管理画面へログインできるか
- WAN側がつながっているか
- Wi-Fi名と暗号化設定
- LAN側IPアドレス
- 設定バックアップの取り方
- ファームウェアバージョン
- 自分の回線方式

特に、IPoE、VLAN、VPN、firewallまわりを触る前には、バックアップを取っておくと安心です。

LuCIではここです。

```txt
System → Backup / Flash Firmware → Generate archive
```

OpenWrtベースのルーターは、できることが多いです。

だからこそ、変更前の状態を残しておくことが大事です。

## よくある質問

### LN6001-JPは普通のWi-Fi 7ルーターと何が違う？

一番の違いは、Wi-Fi 7ルーターとして使い始められる入口を持ちながら、LuCI、SSH、opkg、VLAN、VPN、IPoE/IPv6のようなOpenWrtベースの柔軟性も使えるところです。

普通のルーターより設定できる範囲が広いので、家庭・小さなオフィス・店舗のネットワークを自分の用途に合わせて整理しやすくなります。

### LN6001-JPは自分でOpenWrtを入れる必要がある？

ありません。

LN6001-JPはOpenWrtベースの製品として出荷されるため、一般的な市販Wi-Fiルーターへ自分でファームウェアを書き換える前提ではありません。

購入後は、OpenWrtを導入するところではなく、OpenWrtベースの環境をどう使うかから始められます。

### LN6001-JPは初心者でも使える？

普通のWi-Fiルーターとして使い始めるだけなら、入口はかなり低いです。

ただし、LN6001-JPの面白さは、LuCI、SSH、VLAN、VPN、IPoE/IPv6などへ必要に応じて踏み込めるところにあります。

最初から全部を理解する必要はありませんが、少しずつ設定を見ながら使う気持ちはあったほうが楽しめます。

### LN6001-JPはどんな人に向いている？

Wi-Fi 7ルーターを探しつつ、ゲストWi-Fi、VLAN、VPN、IPoE/IPv6、DNSなどを自分で整理したい人に向いています。

OpenWrtに興味はあるけれど、自分でファームウェアを書き換えるところから始めるのは重い、という人にも入りやすい製品です。

### 逆に、どんな人には向いていない？

スマホアプリだけで完全自動設定したい人、管理画面をほとんど見たくない人、VLANやVPNなどを使う予定がない人には少し多機能です。

ただ速いWi-Fi 7ルーターが欲しいだけなら、もっとシンプルな製品も候補になります。

### Wi-Fi 7対応端末がないと意味がない？

意味がないわけではありません。

5GHz中心の高性能ルーターとして使いながら、あとからWi-Fi 7対応端末や6GHz対応端末を増やしていく考え方もできます。

ただし、MLOや6GHzの恩恵をすぐ受けたい場合は、端末側の対応も確認してください。

### 2.5GbE LANとして使えますか？

LN6001-JPの2.5GbpsポートはWAN側です。

LANポートは1Gbpsが4つです。

有線NASやデスクトップPCを2.5GbEでつなぎたい場合は、この点に注意してください。

## この連載で使っているOpenWrtルーター

この連載では、Linksys Velop WRT Pro 7（LN6001-JP）を使っています。

https://amzn.to/4tVu4yG

LN6001-JPは、国内向けに展開されているOpenWrtベースのWi‑Fi 7ルーターです。一般的な「自分でOpenWrtを書き込むタイプ」と違い、OpenWrtベースで最適化されたファームウェアを搭載した状態で出荷されるため、かなり導入しやすいのが特徴です。

初期セットアップ済みで、基本的には電源を入れてすぐ利用可能。LuCIブラウザ管理画面、SSH、opkg、VLAN、OpenVPN、WireGuard、Tailscaleなど、OpenWrtらしい拡張性も備えています。

特に日本の回線環境向けとして、Linksys公式の「オートIPoE」モジュールが用意されているのが大きなポイントです。OCNバーチャルコネクト、transixなどのIPv4 over IPv6環境でも、LuCIから比較的シンプルに設定できます。

もちろんCLIで細かく調整していく楽しさもありますが、MAP-EやIPIPは設定項目がかなり多いため、「まず普通に安定して使いたい」という場合はオートIPoEを使った方がかなり楽です。

この連載では、実機開発にも関わった立場から、家庭・小規模オフィス・店舗などで“実際どう使うと快適なのか” を中心に、現実的なOpenWrt運用を紹介していきます。
