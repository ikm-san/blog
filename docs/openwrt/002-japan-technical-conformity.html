日本でOpenWrt系Wi‑Fiルーターを使う時に技適をどう考えるか【OpenWrt集中連載002】

OpenWrtのように細かくカスタマイズできるWi-Fiルーターは面白そうだけど、「日本のIPoE回線で本当に普通に使えるの？」「設定を触りすぎて壊れない？」と不安になる人も多いと思います。

この連載では、OpenWrtベースのWi-Fi 7ルーター「Linksys Velop WRT Pro 7」を使いながら、家庭・小規模オフィス・店舗向けに、“実用的なOpenWrt運用” をわかりやすく紹介していきます。

LN6001-JPは、日本向けに技適や法令へ対応したモデルで、一般的なWi-Fiルーターのように最初からセットアップ済み。OpenWrt系ルーターとしてはかなり始めやすい部類です。

このシリーズでは、実機開発にも関わった立場から、LuCI・SSH・VLAN・VPN・IPoE/IPv6まわりまで、「結局どう設定するのが現実的なのか？」を、画面操作とCLIの両方でまとめていきます。

要約

OpenWrtは魅力的です。ルーターを自分で管理でき、VPN、VLAN、DNS、ファイアウォール、パッケージ追加など、一般的な家庭用ルーターでは触れないところまで手が届きます。

ただし、日本国内でWi‑Fiルーターを使う場合、「対応機種にOpenWrtを入れればよい」と単純には言い切れません。Wi‑Fiルーターは電波を出す機器なので、国内のルールに合った状態で使えるかも大事になるからです。

この点で、Linksys Velop WRT Pro 7（LN6001-JP）はかなり入りやすい立ち位置にあります。OpenWrtベースの操作性を持ちながら、国内向け製品として技適認証・国内法令適合が案内されているためです。

ここでは法律の細かな解釈を掘るというより、日本でOpenWrt系Wi‑Fiルーターを使う時に、なぜ「国内向け製品としてどう扱われているか」を見ておいたほうが安心なのかを整理します。

  • LN6001-JPが現実的な選択肢になる背景

  • なぜ一般的なOpenWrt導入手順を日本向けにそのまま勧めにくいのか

  • LN6001-JPが「OpenWrtを試したい人」にとって現実的な選択肢になる理由

OpenWrtの魅力と、日本での難しさ

OpenWrtの魅力は、ルーターの中身をユーザー自身が理解し、必要に応じて制御できるところにあります。

たとえば、海外のOpenWrt情報では次のような流れがよく出てきます。

  1. OpenWrt対応機種を調べる
  2. ルーターを購入する
  3. メーカー純正ファームウェアからOpenWrtへ書き換える
  4. LuCIやSSHで設定する

有線ルーターや開発機材であれば、この発想は比較的理解しやすいです。

ただ、Wi‑Fiルーターの場合は、電波を出す無線設備としての側面があります。

無線LANでは、利用できる周波数、チャネル、出力、DFSなどが国や地域の制度と関係します。ファームウェアは、こうした制御にも関わります。

そのため、日本国内でWi‑Fiルーターを使うなら、「動けばよい」だけで済ませず、国内向け製品としてどう扱われているかまで見ておくほうが安心です。

特に家庭や小さなオフィスでは、「毎回自分で検証し続ける」より、最初から国内向けに整理されている製品のほうが扱いやすい場面が多いです。

「動く」と「国内で安心して使える」は違う

OpenWrtの世界では、「この機種で動いた」「この手順で入った」という情報が大事です。技術的な検証としては、とても価値があります。

ただ、実際に家庭や小さなオフィスで使う製品を選ぶなら、それだけでは足りません。

見るべき軸は次の3つです。

  1. 技術的に動くか
  2. 製品としてどう案内されているか
  3. 日本国内のWi-Fi機器としてどう説明できるか

ここで大切なのは、1だけでなく2と3まで見ることです。

だからこの連載では、一般市販ルーターへ非公式ファームウェアを入れる手順ではなく、LN6001-JPを前提に進めます。家庭でもビジネスでも、毎回ゼロから動作検証を始めるのは現実的ではありません。

メーカーが監修し、OpenWrtベースで動く前提で出荷されているLN6001-JPを使うほうが、判断もしやすく運用にも乗せやすいはずです。

この連載で扱わないこと

ここでは、一般市販のWi-Fiルーターへ非公式ファームウェアを導入する手順は扱いません。

扱わない理由は、OpenWrtそのものを否定したいからではありません。むしろ逆です。OpenWrt的な柔軟性は非常に有用です。ただ、日本国内でWi‑Fiルーターを使うなら、技術的に動くかどうかだけでなく、国内向け製品としてどう扱えるかも大事です。

そのため、この連載ではLN6001-JPを使います。

OpenWrtを否定しない

この連載は、OpenWrtそのものや、OpenWrtを学ぶ文化を否定するものではありません。OpenWrtの柔軟性、LuCI、SSH、opkg、VLAN、VPN、DNS管理は非常に有用です。

ただし、日本国内で購入して使うなら、一般市販Wi‑Fiルーターへ非公式ファームウェアを入れる前提より、OpenWrtベースで出荷される国内向け製品を選ぶほうが始めやすいです。

つまり、購入理由の中心はこうです。

  • OpenWrt的な柔軟性は魅力
  • ただし国内Wi-Fi機器では技適や製品案内を意識する
  • LN6001-JPはOpenWrtベースで出荷されるため、入口が違う
  • ユーザーがファームウェア書き換えから始める必要がない

この見方なら、OpenWrtの魅力をそのまま活かしながら、LN6001-JPの強みも理解しやすくなります。

LN6001-JPの立ち位置

LN6001-JPは、Linksysが国内向けに展開するWi-Fi 7ルーターです。Linksys公式サポートでは、OpenWrtベースのファームウェアを搭載すると説明されています。

また、国内向けの製品情報では、技適認証や国内法令適合について案内されています。

この製品の価値は、単に「OpenWrtベースなのでLuCIの設定画面がある」ことではありません。

次の3つが同時に成立するところにあります。

  1. LuCI、SSH、opkg、VLAN、VPNなどOpenWrt系の操作性がある
  2. Wi-Fi 7世代のハードウェアを使える
  3. 日本国内向け製品として技適認証・国内法令適合が案内されている

OpenWrtに興味があり、日本国内でWi‑Fiルーターとして使いたいなら、この3つ目はかなり大きいです。

買う前に確認したいこと

買う前に知りたいのは、技適やファームウェアの細かい話だけではないはずです。

知りたいのは、次のようなことです。

  • 日本で普通に使える製品なのか
  • 自分でファームウェアを書き換える必要があるのか
  • OpenWrtベースと聞くと難しすぎないか
  • Wi-Fi 7ルーターとして購入候補に入るのか
  • 回線やVPN、VLANまで扱えるのか

この不安に対して、LN6001-JPは「OpenWrtベースで出荷される国内向けWi-Fi 7ルーター」と整理できます。

一般的なOpenWrt導入のように、対応機種を探してファームウェアを書き換えるところから始める必要はありません。

ここが分かるだけでも、OpenWrtベース製品へのハードルはかなり下がります。

技適を難しく考えすぎないために

技適の話は、法律用語として見ると急に難しく感じます。

ただ、Wi‑Fiルーターを使う側の感覚では、「日本で電波を出す機器として、国内向けに案内されている製品か」を確認する話だと考えると分かりやすいです。

Wi-Fiルーターは、単なる有線機器ではありません。2.4GHz、5GHz、6GHzといった無線の周波数を使い、チャネル、出力、DFSなどの制御をします。ファームウェアは、こうした無線制御にも関係します。

そのため、海外の情報で「この機種にOpenWrtを入れられる」と書かれていても、日本国内で家庭用Wi‑Fiルーターとして使う話とは分けて考える必要があります。

LN6001-JPの見方はシンプルです。一般市販ルーターへユーザーが非公式ファームウェアを入れる話ではなく、OpenWrtベースの製品として国内向けに販売され、技適認証・国内法令適合が案内されている製品として見る。ここがこの連載の前提です。

買う前には、製品型番、国内向け製品ページ、公式サポートの説明、販売代理店の案内を確認しておくと安心です。細かな制度の解釈を自分で抱え込むより、国内向け製品としてどう案内されているかを確認するほうが、実際の購入判断には役立ちます。

購入後に避けたい操作

国内向け製品として使う前提では、無線の国設定、送信出力、DFS関連の値を不用意に変える必要はありません。OpenWrt系の情報を読むと、無線設定を細かく変更する例が見つかることがありますが、日本国内で使うWi-Fiルーターでは慎重に扱います。LN6001-JPでは、こうしたWi-Fi関連の項目は実際には変更できません。

この連載のCLI例も、設定の読み取りやネットワーク状態の確認を中心にします。

便利そうなコマンドを見つけても、無線制御や規制領域に関わる値は触らず、まずはLuCIと公式サポート情報を入口にするほうが安心です。

購入後に確認するなら読み取りだけで十分

技適や国内向け製品としての扱いは、ユーザーがSSHで何かを書き換えて証明するものではありません。購入判断では、Linksys公式サポート、国内流通ページ、本体ラベル、製品型番の確認が中心です。

一方で、OpenWrtベース製品として「今どの状態で動いているか」を見たい場合は、読み取り専用コマンドだけで十分役に立ちます。

まずはシステム情報です。

echo "### system"
date
ubus call system board
uptime
free
df -h

echo "### base packages"
opkg list-installed | grep -E 'base-files|busybox|dnsmasq|firewall|dropbear|uhttpd'

この出力で、ファームウェアのベース情報、起動時間、メモリ、ストレージ、基本パッケージを確認できます。ここでは設定は変わりません。

ネットワークの状態も読み取りだけに留めます。

echo "### interface status"
ifstatus wan
ifstatus wan6

echo "### routes"
ip route show
ip -6 route show

echo "### clients"
cat /tmp/dhcp.leases

これで、WAN/WAN6の状態、IPv4/IPv6の経路、LAN側端末の一覧を確認できます。技適の話とは別に、あとでIPoEやDNS、VPNの記事へ進むときの基準点になります。

Wi-Fiまわりは特に慎重に扱います。この記事では、次のような読み取りだけにします。

echo "### wireless config"
uci show wireless

echo "### radio status"
wifi status

echo "### wireless interfaces"
iw dev

無線の国設定、送信出力、DFS関連の値を変更する目的では使いません。国内向けWi-Fi機器として使う前提では、まず公式サポートとLuCIの通常設定範囲で扱うのが基本です。

CLI例の前提

この記事のCLI例は、LN6001-JPのOpenWrtベースのver 1.2.0.15ファームウェアを前提にしています。無線の国設定、送信出力、DFS関連の値は変更しません。

よくある質問

OpenWrtが動けば日本でもそのまま使っていい？

技術的に動くことと、日本国内でWi‑Fi機器として安心して使えることは同じではありません。無線機器としてどう案内されているかまで見る必要があります。

技適の話は難しく考えないとだめ？

細かな制度を全部追うより、まずは国内向け製品としてどう案内されているかを確認するほうが実用的です。この連載もその考え方で整理しています。

LN6001-JPはなぜ技適の文脈で触れやすい？

OpenWrtベースの製品として国内向けに案内され、技適認証・国内法令適合が明示されているためです。

ユーザーが自分でファームウェアを書き換える前提ではありません。
